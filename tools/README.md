# Tool Script Index

The top-level `tools/` folder contains project utilities, not application UI
tools. Application interaction tools live in `local/app/src/yj_studio/tools/`.

Keep these files at their current paths until all references in docs, tests, and
comments are updated.

## Shared Path Helper

- `project_paths.py`: common project-root and data-path resolution for scripts.

## Reservoir Conversion And Checks

- `create_reservoir_3x_numpy.py`: legacy 3x conversion that crops the existing
  seismic-aligned 1x reservoir numpy volumes and repeat-upsamples them.
- `create_reservoir_3x_direct_numpy.py`: server-side 3x conversion that samples
  GRDECL-derived native reservoir arrays directly into the 3x regular runtime
  grid, without repeating the existing 1x seismic-aligned volumes.
- `bake_reservoir_npy.py`: final offline conversion to a cropped, node-aligned
  `6.25 m x 6.25 m x 2 m` numpy runtime grid. It intersects the active GRDECL
  footprint with the seismic bounds and supports metadata-only `--dry-run`.
  The v3 footprint fills only enclosed holes in native logical topology,
  rasterizes representative-depth COORD quadrilaterals, and preserves the true
  exterior boundary.
- `verify_reservoir_625.py`: full-volume validation for the final baked pair,
  including shape, endpoint mapping, support consistency, and value ranges.
- `diagnose_reservoir_625_stripes.py`: identify full-depth nodata stripes and
  distinguish support-mask gaps from property-value gaps.
- `compare_reservoir_625_support.py`: compare candidate lateral support masks
  before committing a full-volume bake.
- `visualize_reservoir_support_candidates.py`: render candidate support
  footprints and representative section boundaries for visual review.
- `compare_reservoir_625_baked.py`: compare actual v2/v3 baked support and
  section content after full-volume generation.
- `visualize_reservoir_625_trios.py`: create five white-cell-grid
  seismic/lithology/porosity comparison examples for each of the three axes.
- `visualize_reservoir_625_global.py`: render full-extent representative
  sections for all three axes, with both screen-sized overviews and
  high-resolution per-cell white-grid images.
- `visualize_reservoir_625_every_12p5m.py`: encode complete three-panel section
  sequences at exact 12.5 m positions for all three axes, plus contact sheets
  and per-frame coordinate manifests.
- `verify_reservoir_625_videos.py`: validate frame counts, dimensions, frame
  rate, and first/last-frame readability for those videos.
- `visualize_reservoir_3x_comparison.py`: render old repeat-3x vs new direct-3x
  lithology/porosity slice comparison PNGs.
- `precompute_reservoir_columns.py`: build native-column geometry/property
  caches consumed by the final baker.
- `probe_grdecl.py`: inspect GRDECL content and dimensions.
- `probe_grdecl_dims.py`: report logical GRDECL dimensions and counts.
- `probe_native_geometry.py`: inspect native cell geometry and spacing.
- `probe_render_vsection.py`, `probe_render_zslice.py`: render native-model
  diagnostic sections under `outputs/diagnostics/`.
- `probe_thin_beds.py`: inspect thin-bed preservation.
- `verify_grdecl_parser.py`: validate GRDECL parser behavior.
- `check_bake_slice.py`, `verify_baked.py`, `verify_native_render.py`: focused
  read-only checks used during reservoir conversion.
- `viz_gate_compare.py`, `viz_gate_diff.py`, `viz_holefill.py`,
  `viz_seismic_reservoir.py`, `viz_trio_set.py`: retained visual diagnostics;
  outputs go to `outputs/diagnostics/`.
- `diagnostics/`: historical read-only investigation helpers.

## SAM3 Setup And Smoke Tests

- `check_sam3_setup.py`: verify SAM3 assets and import readiness.
- `check_sam3_video_deps.py`: verify SAM3 video dependencies.
- `copy_sam3_assets.ps1`: copy SAM3 assets into the expected local locations.
- `smoke_triton_compile.py`: quick Triton compile/cache check.

## Generated Output

- `outputs/diagnostics/`: retained diagnostic images and reports generated by
  conversion/probe scripts.
- `outputs/reports/`: reviewed comparison sets, videos and delivery reports.
- `_sam3_smoke_out/`: generated SAM3 smoke-test output; ignored by git.
- `__pycache__/`: Python bytecode cache; ignored by git.
