# YJ Studio 项目规整计划

> 本文档是经过需求对齐后形成的**项目规整(整理/重构)计划**,不是新功能路线图。
> 配套:总体状态见 [`current_project_status_and_roadmap.md`](current_project_status_and_roadmap.md);
> SAM3/目标/训练主线见 [`target_platform_implementation_roadmap.md`](target_platform_implementation_roadmap.md)(注意其中代码路径已过期,见 §3.4)。

## 0. 定位(已与用户确认)

围绕 **「储层目标:标注 → 对象化 → 2D/3D 展示 → 体积统计 → 训练闭环」** 这一条主线收敛,
删掉与之无关的功能和已废弃的 grid 路径,统一两条割裂的 SAM3 入口。

**已确认的需求边界:**

- 产品定位:专注的「标注 → 训练闭环」平台(不是通用解释套件)。
- 保留的核心数据类型:井、层位、断层、储层模型(岩性/孔隙度);地震体作支撑参考,非主体。
- 目标检测/分割/追踪在**地震体 + 储层模型两套数据上都做**(2026-06-18 决策 C):地震体(稠密、12.5 m、构造细节)上检测构造/圈闭,储层模型(原生列渲染、横向 ~50 m / 纵向细)做属性约束/验证。两者在地震轴坐标下天然对齐。(早先"标注主体仅岩性二值体"已作废。)
- GRDECL / corner-point 储层网格路径:**确认删除**。
- 测量工具:**确认删除**。
- 界面简化力度:**折中**(保留 dock 框架,按主线重排+分组+默认可见性,不重写框架)。

## 1. 核心使用主线(已确认"基本符合")

```
1. 远程加载储层体(岩性二值体 model_lithology / 孔隙度体 model_porosity),地震体作支撑参考
2. 打开 2D 剖面(inline/xline/z)→ 在岩性二值体剖面上框选储层目标
3. AI 面板:SAM3 分割当前剖面 → 候选 mask
4. 跨帧追踪:沿轴传播 → 多帧 mask → GeoTarget(编号一致)
5. 目标面板:确认 / 改类型 / 合并 / 拆分 / 删除
6. 三维重建:目标各帧 mask 堆叠成 (D,H,W) 三维体 → 三维显示(空间形态、连续性)
7. 体积统计:有效体素数 × 单体素体积(dx·dy·dz)→ 目标总体积
8. 导出训练样本 → 服务器训练 → 新模型回推理(闭环)
```

## 4. 储层切片来源:裁剪后的规则 numpy 体(2026-06-18 最终决策)

**决策**:GRDECL 只作为离线源数据，最终运行时仍使用规则 `.npy` 体。离线转换只处理
有效储层覆盖范围与地震体空间范围的交集，并输出 **6.25 m × 6.25 m × 2 m**
的 node-aligned 规则网格，再喂给同一条 `/sam3/jobs`。

- 横向 6.25 m = 地震 12.5 m 的 2× 采样:每个地震面及相邻面中点各有一张储层剖面。
- 纵向 2 m = 地震 10 m 的 5× 采样,接近原生 cell 中位厚度 2.25 m。
- 横向加密用于连续显示、坐标对齐和 SAM3 追踪,不声称增加超过原生约 50 m 的地质信息。
- 转换直接读取 GRDECL-derived native 属性与 ZCORN 几何,不从旧 1x/3x 规则体插值恢复细节。
- 运行时只读最终 numpy 体,不解析 GRDECL,也不重新引入已删除的客户端 grid UI。

**离线转换**:
- 工具:`tools/bake_reservoir_npy.py`。
- 横向:最近原生列;岩性:2 m 深度带区间聚合;孔隙度:最近 `z_center`。
- ACTNUM + ZCORN 决定有效深度范围;横向范围与地震体边界求交。
- 支持域按原生 `(i,j)` 拓扑确定:只填充被有效列包围的内部洞,外边界不动;随后在
  相邻有效列平均活动深度处插值共享 COORD pillar 并栅格化列四边形。不得再用最近列
  距离阈值判定体内/体外,以免产生贯穿剖面的伪 nodata 竖缝。
- 裁剪边界对齐到地震节点并包含两端,避免 `2N`/`2N-1` 半格偏移。
- metadata 记录原点、inclusive 地震索引边界、shape、spacing 和索引映射。

**体积统计**:目标 mask 与最终规则体共用同一网格，按
`有效体素数 × 6.25 m × 6.25 m × 2 m` 统计规则体体积。metadata/config 是采样间隔的
唯一来源；如以后需要 corner-point 精确地质体积，应作为独立统计口径另行实现。

**3D 重建**:目标各帧 mask 直接按最终规则体索引堆叠成 `(D,H,W)`，空间原点和采样间隔
来自最终体 metadata。

**最终规则网格(2026-06-18 锁定)**:储层体素 = **6.25 × 6.25 × 2 m**。
**采样**:岩性 = 区间聚合(深度带内有目标即目标,保薄层);孔隙度 = 最近 z_center;横向 = 最近列。
**SAM3 检测在地震+储层两套上做(方案 C)**:两套数据都走稠密 numpy 数组索引，并通过
metadata 中的原点和比例映射保持同一地震坐标系。运行时不依赖 GRDECL。
