# 圈闭识别研究与技术路线阶段性共识

> 记录日期：2026-06-20
> 文档性质：阶段性研究共识，不作为最终算法方案或固定实现规范。
> 适用范围：YJ Studio 圈闭识别主线，以及浊积体分割、追踪、三维重建和后续模型训练相关工作。

## 1. 文档目的

本文用于记录当前围绕圈闭识别任务形成的核心共识，统一后续讨论中的研究目标、概念边界和技术路线。

现阶段的重点不是立即形成成熟的工业化圈闭识别方案，而是依托已经建立的本地端与服务器端协同框架，完成一次可验证的功能实验，回答以下基础问题：

1. 地质目标能否在二维剖面上被有效识别和分割；
2. 同一目标能否在相邻剖面之间保持稳定身份并连续传播；
3. 多剖面结果能否重建为具有合理空间形态的三维对象；
4. 提示式交互是否适合圈闭识别工作流；
5. 现有方法能否为后续专用模型训练和完整圈闭判定提供基础。

服务器端代码部署、数据迁移和运行数据准备已经基本完成，暂不作为当前研究工作的主要矛盾。当前关注点集中在算法流程、目标表达、交互方式和识别效果本身。

## 2. 项目最初目标

项目的长期目标是形成一个面向圈闭识别的交互式研究平台，使用户能够结合地震体、岩性体、孔隙度体、层位、断层和井资料，对潜在地质目标进行识别、追踪、三维重建、审校和评价。

期望形成的总体流程为：

```text
多源地质数据
  → 文本、框选、点选等交互提示
  → 二维目标搜索与精细分割
  → 跨剖面关联和三维目标重建
  → 人工审校与标签积累
  → 地质条件约束和圈闭合理性评价
  → 专用模型训练与迭代优化
```

其中，模型负责发现和分割疑似目标；构造闭合、盖层、断层封堵、储层属性等地质条件负责进一步判断目标是否构成有效圈闭。最终方案倾向于采用“数据驱动识别 + 地质规则评价”的组合方式，而不是完全依赖单一模型端到端给出圈闭结论。

## 3. 当前研究对象：浊积体

### 3.1 选择原因

现阶段选择浊积体作为初步实验对象，主要基于以下考虑：

- 浊积体在三维空间中通常相对独立，适合作为单独目标进行对象化；
- 二维切片中的边界和主体区域相对容易观察；
- 多剖面分割结果重建后具有较清晰的三维形态，便于人工判断连续性和漂移；
- 目标的分叉、尖灭、连接和消失等现象，可以有效检验跨帧追踪能力；
- 相较复杂构造圈闭或断层封堵圈闭，实验对象更容易控制，适合验证基础流程。

因此，浊积体是一个合理的三维地质目标试验体，能够帮助验证从二维识别到三维对象化的核心技术链。

### 3.2 概念边界

必须明确：浊积体、储集体和圈闭不是同一概念。

浊积体识别主要回答“目标在哪里、边界如何、空间形态怎样”；完整圈闭判定还需要考虑：

- 构造或岩性闭合；
- 上倾方向的封堵条件；
- 盖层条件；
- 断层封堵关系；
- 溢出点和闭合高度；
- 储层物性及保存条件。

因此，当前实验能够直接验证的是：

```text
二维目标分割
→ 跨切片追踪
→ 目标编号一致性
→ 三维重建
→ 体积和连续性统计
```

当前实验尚不能单独证明完整圈闭识别已经实现。更准确的阶段性定位是：

> 以浊积体为代理目标，验证圈闭识别所依赖的目标发现、交互分割、跨剖面关联和三维对象化能力。

## 4. 当前算法验证思路

现阶段不要求一次性形成完整、成熟的解决方案，而是按照由简到繁的方式逐层验证。

### 4.1 第一层：二维分割能力

验证模型能否在单张地震、岩性或孔隙度剖面中识别目标主体，并生成基本合理的边界。

重点观察：

- 是否能够覆盖完整目标；
- 是否误分邻近地质体；
- 小目标、薄层和尖灭区域能否保留；
- 不同输入模态对结果的影响；
- 文本、框和点提示分别能够提供多大帮助。

### 4.2 第二层：跨剖面追踪能力

在一个较可靠的种子剖面上初始化目标，沿 Inline、Xline 或其他方向传播，验证同一目标能否保持稳定身份。

重点观察：

- 目标是否逐渐膨胀或收缩；
- 是否跳转到相邻砂体；
- 目标消失后能否正确处理；
- 分叉、汇合和遮挡位置是否发生 ID 混乱；
- 传播误差是否随距离种子帧增大。

### 4.3 第三层：三维对象化能力

将多帧 mask 按真实空间索引堆叠为三维目标，验证其空间形态、连通性和体积统计是否合理。

重点观察：

- 三维目标是否存在毛刺、孔洞、断裂和虚假桥接；
- Inline 与 Xline 方向重建是否一致；
- 体积、厚度、长宽比和连通体数量是否符合预期；
- 二维局部误差是否在三维空间中被放大。

### 4.4 第四层：圈闭候选评价

在浊积体或储集体识别基本稳定后，再与层位、断层、盖层和储层属性结合，对目标进行圈闭合理性评价。

建议的长期逻辑为：

```text
储集体三维目标
  ∩ 构造闭合范围
  ∩ 有效储层属性
  ∩ 盖层与断层封堵约束
  → 圈闭候选
```

圈闭评分应尽量保持可解释性，可拆分为结构闭合、储层质量、封堵条件、三维连续性和模型可信度等组成部分。

## 5. 采用 SAM3 架构的原因

### 5.1 需求不只是固定二值分割

后续期望支持的圈闭识别流程包含多种交互方式：

1. **文本提示**：指定某一类圈闭或地质目标，在整张图中搜索并识别全部候选实例；
2. **框选提示**：限定候选区域，在局部范围内完成更精细的实例分割；
3. **点选提示**：通过正点、负点或后续修正点调整模型结果；
4. **跨剖面传播**：将已确认目标传播到其他切片，并保持目标编号一致；
5. **人工审校**：对模型结果进行确认、修正、合并、拆分和回流训练。

普通固定类别二值分割模型通常只解决“输入一张图，输出一个 mask”，不能天然覆盖上述完整交互需求。

### 5.2 SAM3 的参考价值

SAM3 的整体架构已经包含或能够支持：

- 图像特征编码；
- 文本提示编码；
- 框等几何提示编码；
- 多实例目标搜索；
- 实例分割；
- 视频或序列目标追踪；
- 交互式修正；
- 文本、框、mask 等多种监督形式的训练。

因此，SAM3 的价值不只在于原生权重的分割精度，还在于它提供了一套与目标需求高度相似的提示式识别框架。

当前采用 SAM3 的主要目的包括：

- 快速验证文本、框选和跨帧追踪工作流；
- 在专用训练数据不足时提供可用的初始能力；
- 辅助生成和修正标签；
- 作为后续领域微调的基础模型；
- 作为轻量化专用模型的教师模型和性能参考。

### 5.3 对原生权重的认识

原生 SAM3 权重主要来自自然图像和通用视觉任务，与地震、岩性和储层属性数据存在明显领域差异。因此：

- 原生权重适合用于功能原型和零样本基线；
- 文本编码器未必能直接理解专业地质概念；
- 地学切片的纹理、尺度、颜色和空间关系需要领域适配；
- 原生效果不理想不能直接否定提示式架构本身；
- 是否继续使用 SAM3，应根据微调后效果、数据规模和工程成本判断。

## 6. SAM3 训练机制的阶段性认识

根据项目内置源码，SAM3 图像模型主要包含视觉骨干、文本编码器、几何提示编码器、Transformer 检测模块、分割头和交互预测模块；视频模型在此基础上增加目标追踪与记忆传播能力。

其训练框架能够使用：

- 类别文本查询；
- 实例边界框；
- 实例 mask；
- 文本与视觉提示组合；
- 分类与目标存在性损失；
- 边界框回归损失；
- Mask focal loss；
- Dice loss；
- 图像和视频样本。

这说明后续可以围绕专业圈闭类型构造文本、框和 mask 监督，对部分模块或整个模型进行领域微调。

现阶段不建议直接进行大规模全参数训练。更稳妥的实验顺序是：

1. 使用原生权重建立文本、框选和追踪基线；
2. 冻结大部分视觉骨干，优先训练分割头、检测/匹配头和几何提示相关模块；
3. 对文本侧使用较小学习率，先验证有限的固定专业类别短语；
4. 随标签数量和多样性增加，再逐步解冻视觉骨干后部；
5. 将训练后模型与原生权重进行固定测试集对比。

## 7. 标签样本与训练数据设想

已经补充的标签样本将作为后续领域微调和轻量化训练的基础。标签不应只保存最终二值 mask，还应尽量保留完整实验上下文。

建议每个样本至少记录：

- 输入数据类型和体数据 ID；
- 切片轴、索引和空间网格信息；
- 目标类型及使用的专业文本；
- 实例 mask 和 bbox；
- 所属三维目标 ID；
- 是否由人工确认或修正；
- 使用的提示类型和提示内容；
- 模型版本和预处理配置；
- 样本难度、边界可信度或失败原因。

为了训练文本搜索能力，同类目标应尽量完整标注。若一张图中的同类实例存在大量漏标，模型可能把未标目标错误学习为背景。

为了训练框选和点选交互能力，可以从人工 mask 自动构造：

- 带随机扰动的候选框；
- 目标内部正点；
- 目标外部和易混淆区域负点；
- 边界附近的修正点；
- 多轮交互的提示序列。

训练集、验证集和测试集应按空间块或完整三维目标划分，不能随机拆分相邻切片，以避免同一目标同时出现在训练和测试数据中。

## 8. 后续轻量化专用模型设想

### 8.1 轻量化的必要性

如果后续任务类别、输入形式和交互方式逐渐稳定，SAM3 的完整通用能力可能带来较高的显存、推理时间和部署复杂度。届时可以考虑训练面向圈闭识别的轻量化专用模型。

但轻量化不应只保留普通二值分割能力。如果最终产品仍需支持文本、框选和点选，那么替代模型也需要保留相应的提示接口。

### 8.2 可行的轻量化路径

可考虑的结构为：

```text
较小的视觉编码器
+ 轻量文本或类别编码器
+ 框/点提示编码器
+ 轻量 mask decoder
+ 可选的序列传播模块
```

轻量模型可以利用微调后的 SAM3 进行知识蒸馏，学习其：

- 候选区域搜索能力；
- 提示响应能力；
- 实例 mask；
- 中间视觉特征；
- 跨帧传播结果。

另一种可行方式是采用分层模型：

```text
轻量检测/语义模型负责全图搜索
→ 小型交互分割模型负责框和点修正
→ 独立追踪模块负责跨剖面传播
```

这种拆分方式比将全部能力压入单一模型更容易训练和维护。

### 8.3 决策时机

是否正式替换 SAM3，应在获得以下实验数据后再决定：

- 原生 SAM3 的文本、框选和点选基线；
- 领域微调后的准确率和召回率；
- 推理速度、显存和吞吐量；
- 标签规模和类别数量；
- 不同交互方式带来的实际收益；
- 轻量模型相对 SAM3 的精度损失；
- 服务器端运行成本和真实使用频率。

因此，当前共识不是“长期必须使用 SAM3”，也不是“尽快删除 SAM3”，而是：

> 先利用 SAM3 验证提示式识别架构并积累高质量标签，再以实验结果决定继续微调、蒸馏轻量化，还是替换为分层专用模型。

## 9. 当前实现与目标之间的已知差距

目前已有工程流程能够支持远程二维分割、框选追踪、目标对象化、三维 mask 重建、体积统计和训练数据导出，但仍存在以下研究性缺口：

- 当前点选在单帧流程中主要被转换为小框，不是真正的正负点迭代修正；
- 追踪过程中尚未形成“在中间帧点选修正后重新传播”的完整交互闭环；
- 文本提示尚未经过地学语义训练；
- 多源输入目前主要被转换为灰度伪 RGB，尚未充分利用地震、岩性和孔隙度联合信息；
- 当前三维结果主要来自单轴二维 mask 堆叠，缺少多轴一致性约束；
- 尚未建立固定真值集及二维、追踪、三维和地质合理性评价指标；
- 现有浊积体或砂体结果尚未与构造闭合、盖层和断层封堵条件形成正式联合判定。

这些问题属于后续实验和实现需要逐步解决的内容，不影响当前使用 SAM3 验证总体交互流程的合理性。

## 10. 已形成的阶段性结论

经过当前讨论，形成以下共识：

1. **研究对象选择合理**
   浊积体具有相对独立、边界较清晰和三维可视化直观等特点，适合作为圈闭识别技术链的初步验证对象。

2. **当前验证目标应保持准确**
   当前直接验证的是三维地质目标的识别、追踪和重建能力，而不是完整的圈闭成因和有效性判定。

3. **先证明流程可行，再追求成熟方案**
   现阶段重点是确认算法链是否能够运行、交互方式是否有效、三维结果是否具有基本合理性，不要求立即达到工业应用水平。

4. **任务不是普通二值分割**
   长期需求包含文本全图搜索、框选精细分割、点选修正和跨剖面传播，因此需要提示式、多实例、可交互的识别架构。

5. **保留 SAM3 架构具有合理性**
   SAM3 的文本、几何提示、实例分割和视频追踪能力与目标需求高度一致，适合作为功能原型、初始模型、标签辅助工具和后续教师模型。

6. **原生权重只是起点**
   原生权重与地学数据存在领域差异，需要通过专业标签、输入表达和领域微调判断其真实上限。

7. **轻量化是后续选项，而非当前前置条件**
   在交互需求和模型效果尚未得到定量验证前，不宜过早将系统收缩为普通轻量二值分割模型。

8. **轻量化应尽量保留交互能力**
   后续专用模型应根据实际需求保留文本、框、点中的必要能力，可通过蒸馏、模块替换或分层模型降低复杂度。

9. **圈闭判定需要模型与地质规则结合**
   模型主要负责目标搜索和边界分割；完整圈闭识别需要进一步融合层位、断层、盖层、储层属性和闭合条件。

10. **标签和评价体系是下一阶段基础**
    后续模型选择、微调和轻量化决策必须建立在高质量标签、空间独立的数据划分和可重复的定量评价之上。

## 11. 建议的后续讨论顺序

后续可按以下顺序继续细化：

1. 明确第一轮实验使用的数据模态、目标类别和测试样本；
2. 定义文本搜索、框选分割、点选修正三个独立实验；
3. 建立二维、追踪和三维重建的评价指标；
4. 检查现有标签格式是否满足 SAM3 训练要求；
5. 设计真正的正负点交互和中间帧修正流程；
6. 制定 SAM3 领域微调的最小可行方案；
7. 比较原生权重、微调权重和轻量模型；
8. 将稳定的浊积体结果接入构造闭合和封堵条件评价。
